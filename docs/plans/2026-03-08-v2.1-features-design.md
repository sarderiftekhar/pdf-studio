# v2.1 Feature Design — Bootstrap, Barcode/QR, Thumbnails, TOC & Bookmarks

Date: 2026-03-08
Branch: `feature/v2.1-bootstrap-barcode-thumbnail-toc`

## 1. Bootstrap CSS Support

**Decision:** Mutually exclusive with Tailwind. User picks one per render.

### Pipeline Change

```
Blade → [TailwindCompiler OR BootstrapInjector] → CssInjector → PdfRenderer
```

`RenderPipeline` selects the CSS step based on `RenderContext::$cssFramework` (set by `->bootstrap()` or `->tailwind()`, or config default).

### API

```php
Pdf::view('invoice')->bootstrap()->render();
Pdf::view('invoice')->tailwind()->render(); // explicit
```

### Config

```php
'css_framework' => 'none', // 'tailwind', 'bootstrap', 'none'
```

### Files

| File | Purpose |
|------|---------|
| `src/Pipeline/BootstrapInjector.php` | Reads bundled CSS, sets `compiledCss` |
| `resources/css/bootstrap.min.css` | Bootstrap 5.3 minified |
| `src/Pipeline/RenderPipeline.php` | Updated to select CSS step |
| `src/PdfBuilder.php` | `->bootstrap()`, `->tailwind()` methods |
| `src/DTOs/RenderContext.php` | `$cssFramework` property |
| `src/PdfStudioServiceProvider.php` | Register BootstrapInjector |
| `config/pdf-studio.php` | `css_framework` key |

## 2. Barcode & QR Code Blade Directives

**Decision:** Pure PHP SVG generation. Works across all drivers.

### Dependencies (suggested)

- `picqer/php-barcode-generator` — SVG barcodes
- `chillerlan/php-qrcode` — SVG QR codes

### Blade Directives

```blade
@barcode('CODE128', $value, ['width' => 2, 'height' => 50])
@qrcode($url, ['size' => 150])
```

Output: inline SVG.

### Files

| File | Purpose |
|------|---------|
| `src/Barcode/BarcodeGenerator.php` | Wraps picqer, returns SVG |
| `src/Barcode/QrCodeGenerator.php` | Wraps chillerlan, returns SVG |
| `src/PdfStudioServiceProvider.php` | Register `@barcode` and `@qrcode` directives |
| `config/pdf-studio.php` | `barcode` and `qrcode` default settings |

### Config

```php
'barcode' => [
    'default_type' => 'CODE128',
    'default_width' => 2,
    'default_height' => 50,
],
'qrcode' => [
    'default_size' => 150,
    'error_correction' => 'M',
],
```

## 3. PDF Thumbnails

**Decision:** Dual strategy — Imagick for arbitrary PDFs, Chromium screenshot for own renders. Auto-fallback.

### API

```php
// From existing PDF file
Pdf::thumbnail('/path/to/doc.pdf', page: 1, width: 300, format: 'png');

// From a render (Chromium screenshot)
Pdf::view('invoice')->data($data)->thumbnail(width: 300, format: 'webp');
```

### Files

| File | Purpose |
|------|---------|
| `src/Thumbnail/ThumbnailGenerator.php` | Main class, picks strategy |
| `src/Thumbnail/ImagickStrategy.php` | Uses Imagick extension |
| `src/Thumbnail/ChromiumStrategy.php` | Uses Browsershot screenshot |
| `src/Thumbnail/ThumbnailResult.php` | DTO: `content()`, `save()`, `base64()`, `mimeType()` |
| `src/Contracts/ThumbnailContract.php` | Interface |
| `src/PdfBuilder.php` | `->thumbnail()` method |
| `src/Facades/Pdf.php` | `Pdf::thumbnail()` static |
| `config/pdf-studio.php` | `thumbnail` config section |

### Config

```php
'thumbnail' => [
    'strategy' => 'auto', // 'auto', 'imagick', 'chromium'
    'default_width' => 300,
    'default_format' => 'png',
    'quality' => 85,
],
```

### Output Formats

PNG, JPEG, WebP.

## 4. Auto Table of Contents & Bookmarks

**Decision:** Two-pass rendering (Chromium only for page numbers). Heading detection: auto-scan h1-h6, opt-out with `data-toc="false"`, opt-in mode via config.

### API

```php
Pdf::view('report')->data($data)
    ->withTableOfContents()
    ->withBookmarks()
    ->render();

// With options
Pdf::view('report')->data($data)
    ->withTableOfContents(depth: 3, title: 'Contents')
    ->render();
```

### Two-Pass Flow

1. Render HTML via Blade + CSS pipeline
2. Inject JS to extract heading positions + page numbers from Chromium
3. Generate TOC HTML from extracted entries
4. Prepend TOC page to document HTML
5. Re-render final PDF with TOC + bookmark outline

### Heading Detection

- Default (`auto`): all `<h1>`-`<h6>` tags
- Exclude: `<h2 data-toc="false">Skip This</h2>`
- Explicit mode: only `<h2 data-toc>Include This</h2>`

### Files

| File | Purpose |
|------|---------|
| `src/TableOfContents/TocExtractor.php` | Parse headings, get page numbers via Chromium JS |
| `src/TableOfContents/TocRenderer.php` | Generate TOC HTML page |
| `src/TableOfContents/BookmarkGenerator.php` | Generate PDF outline |
| `src/DTOs/TocEntry.php` | DTO: level, text, pageNumber, anchorId |
| `src/DTOs/TocOptions.php` | DTO: depth, title, mode |
| `resources/views/toc.blade.php` | Publishable TOC template |
| `src/PdfBuilder.php` | `->withTableOfContents()`, `->withBookmarks()` |
| `src/DTOs/RenderOptions.php` | TOC and bookmark flags |
| `config/pdf-studio.php` | `toc` config section |

### Config

```php
'toc' => [
    'enabled' => false,
    'depth' => 6,
    'title' => 'Table of Contents',
    'mode' => 'auto', // 'auto' or 'explicit'
    'view' => 'pdf-studio::toc',
],
```

### Limitation

Two-pass TOC with accurate page numbers requires Chromium driver. dompdf/wkhtmltopdf can generate bookmarks but not page-accurate TOC.
